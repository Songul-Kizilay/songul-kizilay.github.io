Lab: User ID controlled by request parameter with data leakage in redirect

Bu lab, yönlendirme (redirect) yanıtının gövdesinde hassas bilgilerin sızdırıldığı bir access control zafiyeti içeriyor.

Labı çözmek için carlos kullanıcısına ait API key bilgisini elde edip çözüm olarak göndermem gerekiyor.

Uygulamaya şu bilgilerle giriş yapılabiliyor:
wiener : peter

İlk olarak wiener : peter bilgileriyle uygulamaya giriş yaptım.

Giriş yaptıktan sonra siteyi biraz gezdim, ürünler arasında dolaştım ve carlos kullanıcısına ait herhangi bir bilgiye doğrudan ulaşabilir miyim diye baktım. Ancak bu aşamada carlos’a ait açık bir veri bulamadım.

Daha sonra kendi hesabıma ait My Account sayfasını inceledim. Sayfaya giderken kullanılan URL şu şekildeydi:

/my-account?id=wiener

Burada kullanıcı bilgisinin doğrudan request parametresi ile kontrol edildiğini fark ettim. Bunun üzerine URL’deki wiener değerini carlos ile değiştirerek isteği gönderdim.

Tarayıcı üzerinden gönderdiğimde uygulama beni hemen çıkış yapmış gibi davrandı ve login sayfasına yönlendirdi. İlk bakışta erişim kontrolü düzgün çalışıyormuş izlenimi veriyordu.

Ama aynı isteği Burp Repeater üzerinden tekrar gönderdiğimde farklı bir durum gördüm. Uygulama, redirect gerçekleşmeden önce kısa bir süreliğine carlos’un hesabına ait sayfayı response body içerisinde döndürüyordu.

Bu ilk response içinde carlos kullanıcısına ait API key bilgisi açık şekilde yer alıyordu. Yani uygulama her ne kadar beni çıkışa yönlendirse de, hassas veri zaten response içerisinde sızdırılmış oluyordu.Carlos’un API key’ini alıp çözüm olarak gönderdim ve labı bu şekilde tamamladım.
