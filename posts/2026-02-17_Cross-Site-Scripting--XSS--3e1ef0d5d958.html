Cross-Site Scripting (XSS)

XSS (Cross-Site Scripting) Nedir

HTML Injection zafiyetleri çoğu zaman bir üst levele çıkartılabilir.

Çünkü HTML yerine JavaScript çalıştırabilirsek artık olay "görsel oynama" değil, hesap ele geçirme seviyesine gelir.

XSS'in tanımı:

Kullanıcı girdisi üzerinden tarayıcıda JavaScript çalıştırılmasıdır.

HTML Injection ile XSS arasındaki fark

Bu ikisi sınavlarda çok sorulur. Kısa ve net:

  • HTML Injection → HTML kodu enjekte edersin (sayfa bozulur, form eklenir)
  • XSS → JavaScript enjekte edersin (cookie çalma, token çalma, işlem yaptırma)

XSS'in 3 ana türü

Reflected XSS

Kullanıcı input'u server'a gider, response içinde geri döner.

Örnek:

  • arama sayfası
  • hata mesajları

Genelde saldırgan bir link üretir ve kurbana tıklatır.

Stored XSS

Kullanıcı input'u DB'ye kaydolur ve başka kullanıcılar sayfayı açınca çalışır.

Örnek:

  • postlar
  • yorumlar
  • profil açıklaması

En tehlikelisi budur çünkü:

kurbanın bir linke tıklamasına bile gerek kalmaz.

DOM XSS

Input server'a gitmeden tarayıcıda işlenir ve DOM'a yazılır.

Örnek:

  • URL parametreleri
  • hash fragment
  • sayfa title
  • username alanı

DOM XSS örneği (cookie gösteren payload)

HTML Injection örneğinde input sanitization yoktu.
Bu yüzden aynı sayfa XSS'e de açık olabilir.

Payload:

#"><img src=/ onerror=alert(document.cookie)>

Ne oluyor

  • Tarayıcı bizim input'u DOM'a ekliyor
  • img tag'i oluşuyor
  • src=/ çalışmayınca hata veriyor
  • onerror event'i tetikleniyor
  • JS çalışıyor
  • cookie alert ile ekrana geliyor

Saldırgan bunu ne için kullanır

XSS sadece "alert(1)" değildir.

Gerçek hedefler:

  • session cookie çalmak
  • CSRF gibi aksiyonları kullanıcı adına yaptırmak
  • hesap ele geçirmek
  • keylogger
  • kullanıcıyı fake login sayfasına yönlendirmek (phishing)

Cookie çalma tuzağı

Modern sitelerde cookie'lerin çoğu:

  • HttpOnly olarak ayarlanır

Bu ne demek

document.cookie ile okunamaz

Ama bu "XSS önemsiz" demek değildir.

Çünkü XSS ile hâlâ şunları yapabilirsin:

  • kullanıcı adına istek atmak (session zaten tarayıcıda)
  • DOM üzerinden token çalmak
  • hesap içi işlemleri tetiklemek
  • phishing yapmak

Yani XSS her zaman tehlikelidir.

XSS'in en çok göründüğü yerler

Aşağıdakiler "XSS davetiyesi" gibi:

  • innerHTML
  • outerHTML
  • document.write()
  • insertAdjacentHTML()
  • eval()
  • setTimeout("...")
  • setInterval("...")

Bu fonksiyonları görürsen aklına direkt XSS gelsin.

Güvenli alternatifler

Savunma tarafında yapılması gerekenler:

  • textContent
  • innerText
  • createElement() + appendChild()
  • output encoding (escape)

Kısaca:

HTML olarak basma, text olarak bas.

14) XSS Test Payload Cheat Sheet (HTB / PortSwigger)

Bu payload'lar hızlı test için altın değerinde:

HTML Injection testi

<h1>test</h1>

XSS testi (en basit)

<script>alert(1)</script>

Çoğu yerde çalışan (img trick)

<img src=x onerror=alert(1)>

Cookie göstermek (ders payload'ı)

#"><img src=/ onerror=alert(document.cookie)>

DOM XSS kaynakları

DOM XSS'in en yaygın kaynakları:

  • location.hash
  • location.search
  • location.href
  • document.URL
  • document.referrer

DOM XSS sink

Şunlar "sink"tir, yani input buraya girerse patlar:

  • element.innerHTML = userInput
  • document.write(userInput)
  • insertAdjacentHTML(userInput)

Sonuç: Bu iki zafiyetin özeti

HTML Injection ve XSS, web güvenliğinin "giriş kapısı" gibi.

HTML Injection çoğu zaman şu anlama gelir:

"Kullanıcı sayfayı değiştirebiliyor."

XSS ise şunu anlatır:

"Kullanıcı tarayıcıda JavaScript çalıştırabiliyor."

Ve bu ikinci cümle, saldırgan için neredeyse her zaman "oyun bitti" demektir.
